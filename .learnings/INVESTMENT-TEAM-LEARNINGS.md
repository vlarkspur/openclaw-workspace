## 团队学习原则

1. **错误不隐瞒** —— 每个错误都记录
2. **纠正要升级** —— 用户的纠正反馈给全团队
3. **经验要共享** —— 一个成员学到的，其他成员也要知道
4. **每周要总结** —— Elon 周报包含团队成长内容

---

## 学习升级流程

```
用户纠正/团队犯错
       ↓
记录到 .learnings/LEARNINGS.md 或 ERRORS.md
       ↓
判断是否重要：
├─ 是 → 升级到对应团队 SKILL.md
├─ 影响多个团队 → 升级到 AGENTS.md
└─ 行为准则 → 升级到 SOUL.md
       ↓
每周五 Elon 周报汇总团队成长
```

---

## 升级规则

满足以下任一条件 → 必须升级：

| 条件 | 升级目标 |
|------|----------|
| 优先级 = high 且 状态 = resolved | 对应团队 SKILL.md |
| 重复出现 ≥ 3 次 | AGENTS.md（系统流程） |
| 用户明确说"记住这个" | SOUL.md（行为准则） |
| 影响投资决策 | 相关团队 SKILL.md |
| 涉及数据准确性 | 投资监控小弟 SKILL.md |
| 涉及风控触发 | 风控监控小弟 SKILL.md |
