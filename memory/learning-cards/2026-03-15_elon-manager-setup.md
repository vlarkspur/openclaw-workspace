# 学习卡片：Elon 部门经理搭建

**日期：** 2026-03-15  
**主题：** 搭建学习部门经理 Elon（统筹汇报）  
**耗时：** 约 15 分钟

---

## 遇到的问题

学习系统已经有导师、监工、秘书、考官，但缺少一个统筹全局的角色。小溪需要有人定期汇报学习进展，给出方向性建议。

---

## 解决方案

### 第一步：创建 Elon 技能

位置：`skills/learning-manager-elon/`

包含文件：
- `SKILL.md` - Elon 的核心逻辑和管理规则
- `references/report-template.md` - 经理汇报模板
- `references/management-rules.md` - 管理规则和干预阈值

### 第二步：配置定时任务

在 `HEARTBEAT.md` 中添加：
- 每周日 20:00 触发
- 收集秘书、考官、监工的报告
- 生成经理视角的周报
- 飞书发送给小溪

### 第三步：定义管理策略

**Elon 的人设：**
- 直接、犀利、不迎合
- 数据导向、结果导向
- 该鼓励时不吝啬表扬
- 像马斯克一样：疯狂但靠谱

**干预阈值：**
- 连续 3 天未学习 → 温和提醒
- 连续 5 天未学习 → 正式警告
- 连续 7 天未学习 → 严重警告 + 约谈
- 测试正确率 <50% 连续 2 周 → 调整策略建议

### 第四步：汇报结构

- 本周概览（学习天数、主题数、正确率等）
- 亮点（表扬）
- 问题（建设性建议）
- 下周学习建议（P0/P1/P2 优先级）
- 长期趋势（正确率、投入趋势）
- 经理寄语

---

## 关键命令/代码

```bash
# Elon 汇报存放位置
memory/learning-reports/YYYY-Www-elon-report.md

# 手动触发 Elon 汇报
"Elon，汇报本周进展"
```

## 踩过的坑

- 一开始想让 Elon 直接管理所有 Agent，但 OpenClaw 的 Agent 之间不能直接通信
- 解决方案：Elon 通过读取文件（秘书周报、考官报告）来"收集"信息

---

## 下次遇到同样问题，第一步做什么

先定义汇报结构和触发时间，再设计管理规则和干预阈值。

- 使用 HEARTBEAT.md 定时触发
- 汇报风格参考 Max Black 模式（毒舌但暖心）
