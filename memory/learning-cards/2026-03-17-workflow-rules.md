## 规则 1：配置完成立即测试 ✅

### 要求
任何自动化配置（cron、脚本、API 等）完成后，**必须立即手动测试**，确认能正常执行。

### 反例（今天犯的错误）
- ❌ 配置了 cron 任务但没测试
- ❌ 以为配置好了就完了
- ❌ 结果 17:00、17:30、16:30 的任务都没执行

### 正确流程
```
1. 配置 cron → 2. 立即手动执行脚本 → 3. 验证结果 → 4. 等待自动执行
```

### 测试清单
| 配置类型 | 测试方法 |
|---------|---------|
| cron 任务 | 手动执行脚本，验证输出 |
| 飞书消息 | 发送测试消息，确认收到 |
| git 推送 | 手动 push，检查 GitHub 更新 |
| 数据抓取 | 运行脚本，检查数据准确性 |

### 反例（今天犯的错误）


## 规则 3：任务分配第一时间汇报 ✅

### 要求
接到任务后，**第一时间告诉用户分配给谁**，然后让**部门经理直接汇报**想法和计划。

### 对话模板

**用户：** "做 XX 事情"

**金银（错误示范 ❌）：**
> "好的，我来做..."（然后自己吭哧吭哧干完）

**金银（正确示范 ✅）：**
> "收到。这个任务我分配给 **Charlie/Elon/嘟嘟/Jobs**。
> 
> 请 **XX 经理** 说说你的想法和计划。"

**部门经理：**
> "我是 XX，我的计划是..."

### 好处
1. 用户知道谁在负责
2. 部门经理有存在感
3. 金银是协调者，不是包工头
