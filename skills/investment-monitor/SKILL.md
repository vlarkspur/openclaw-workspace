### 数据标注规范

**每条数据必须标注：**

```markdown
**数据项：** 股东户数
**数值：** 123,456 户
**来源：** 同花顺 iFind
**时间：** 2026-03-14
**可信度：** 实时数据 ⭐⭐⭐⭐⭐
**交叉验证：** 东方财富（一致）
```

### 可信度分级

| 等级 | 说明 | 用途 |
|------|------|------|
| ⭐⭐⭐⭐⭐ | 实时数据、官方披露 | 可直接用于决策 |
| ⭐⭐⭐⭐ | 滞后<7 天、可靠来源 | 可用于参考 |
| ⭐⭐⭐ | 滞后>7 天、单一来源 | 需交叉验证 |
| ⭐⭐ | 估算数据、用户生成 | 仅供参考 |
| ⭐ | 模拟数据、未验证 | 不可用于决策 |

## 交叉验证规则

### 关键数据定义

符合以下任一条件即为关键数据：
- 直接影响买卖决策（股东户数、归母净利润、融资余额）
- 触发风控阈值
- 涉及金额 > 1 万元或仓位 > 5%

### 验证要求

**关键数据必须：**
1. 至少 2 个独立来源
2. 来源之间无关联（如都是爬取同一数据源则不算独立）
3. 差异>5% 时必须人工核查

### 验证流程

```
获取数据 → 标注来源 → 查找第二来源 → 对比
     ↓
差异<5% → ✅ 通过，记录两个来源
差异>5% → ⚠️ 警告，查找第三来源或人工核查
```

## 与 Charlie 配合

**工作流程：**
1. 投资监控小弟抓取数据
2. 标注来源和可信度
3. Charlie 审核数据质量
4. 通过后用于投资决策

**Charlie 常见质问：**
- "数据来源标注了吗？"
- "交叉验证了吗？"
- "这个数据是实时的还是滞后的？"
- "只有一个来源？不行，再找一个"
