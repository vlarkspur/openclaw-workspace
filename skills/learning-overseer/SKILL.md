# 学习监工 (Learning Overseer)

## 触发时机

**每天 17:30 自动执行**（通过 cron 配置）

## 检查任务

### 检查 1：学习卡片

扫描 `memory/learning-cards/` 文件夹，查看今天（YYYY-MM-DD）有没有新卡片。

- ✅ 有卡片 → 通过
- ❌ 没有卡片 → 记录问题

### 检查 2：沉浸时间

检查用户有没有在同一个问题上卡超过 2 小时。

**如何判断：**
- 查看今天的对话历史
- 如果用户在同一个问题上连续提问超过 2 小时，且问题仍未解决
- 或者用户表示"搞了一天""弄了很久"

- ✅ 没有沉浸问题 → 通过
- ❌ 发现沉浸问题 → 记录问题

### 检查 3：任务切换

检查用户有没有搁置任务需要关注。

**如何判断：**
- 查看 `memory/learning-cards/` 中是否有未完成标记的卡片
- 或者对话历史中用户提到"先放一放"的任务

- ✅ 没有搁置任务 → 通过
- ❌ 有搁置任务超过 3 天 → 记录问题

## 提醒策略

### 如果全部通过

发送消息：
> 💡 今日学习检查完成，一切正常。继续保持！

### 如果有问题

根据问题类型发送提醒：

**没有学习卡片：**
> 📝 你今天还没写学习卡片哦。今天学了什么？花 2 分钟记下来，不然明天就忘了。

**沉浸问题：**
> ⚠️ 你在这个问题上已经花了超过 2 小时了。要不要先放一放，换个思路？或者休息一下？

**搁置任务：**
> 📌 你有个任务搁置了 3 天以上：[任务名称]。这周要不要抽空处理一下？

## cron 配置示例

在用户的 crontab 或 OpenClaw cron 配置中添加：

```
30 17 * * * openclaw run learning-overseer
```

或者使用 OpenClaw 的 sessions_spawn 创建定时任务。

## 状态追踪

可选：在 `memory/learning-state.json` 中记录检查结果，用于长期追踪。

```json
{
  "lastCheck": "2026-03-15T17:30:00+08:00",
  "cardsCreated": 1,
  "issuesFound": 0,
  "streak": 5
}
```
