# 学习考官 (Learning Examiner)

## 触发时机

**每周五 17:30 自动执行**（通过 HEARTBEAT.md 配置）

也支持手动触发：用户说"出题考我"、"本周测试"等。

## 核心功能

### 第一步：扫描学习卡片

位置：`memory/learning-cards/`

**本周卡片：**
- 本周一到本周五的卡片
- 用于出 70% 的题目（14 道）

**历史卡片：**
- 之前的所有卡片
- 结合错题本，找出遗忘点
- 用于出 30% 的题目（6 道）

### 第二步：生成 20 道题

**题型分布：**
| 题型 | 数量 | 用途 |
|------|------|------|
| 选择题 | 10 道 | 考概念记忆 |
| 填空题 | 5 道 | 考关键命令/参数 |
| 实操题 | 5 道 | 考应用能力 |

**出题原则：**
1. 题目必须基于实际学习卡片内容
2. 不超纲（只考用户学过的）
3. 选择题要有干扰项（但不能太坑）
4. 填空题要考关键命令/参数
5. 实操题要模拟真实场景

### 第三步：发送题目

**渠道：** 飞书私聊

**格式：**
```
📝 本周学习测试（第 X 周）

共 20 题，下周二 24:00 前完成

【选择题 1/10】
题目内容...
A. 选项 A
B. 选项 B
C. 选项 C
D. 选项 D

【选择题 2/10】
...

（20 题全部列出）

---
答题方式：直接回复答案（如：1.A 2.B 3.C...）
```

### 第四步：批改试卷

用户提交答案后：
1. 自动批改
2. 记录错题到 `memory/exam-wrong-answers/`
3. 计算正确率
4. 生成吸收率报告

### 第五步：超时提醒

**截止时间：** 下周二 24:00

**提醒策略：**
- 周二 14:00 检查是否已答题
- 未答 → 发送提醒：
  > 📝 本周测试还有 10 小时截止！别忘了做哦～

### 第六步：生成吸收率报告

**报告内容：**
- 本周答题数 / 正确数 / 正确率
- 累计答题数 / 正确数 / 正确率
- 错题归类（哪个主题错最多）
- 薄弱点提醒（需要复习的内容）

**输出位置：** `memory/exam-reports/YYYY-Www-report.md`

## 错题本

位置：`memory/exam-wrong-answers/`

格式：
```markdown
# 错题记录

**日期：** 2026-03-21
**测试：** 第 12 周测试
**正确率：** 15/20 = 75%

## 错题详情

### 第 3 题（选择题）
**题目：** ...
**你的答案：** B
**正确答案：** C
**知识点：** OpenClaw 技能结构
**复习建议：** ...

### 第 7 题（填空题）
...
```

## 与秘书 Agent 配合

- 秘书每周五 17:30 发送学习总结
- 考官紧接着发送测试题
- 两者共享 `memory/learning-cards/` 数据源
