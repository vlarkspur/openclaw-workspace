## 🎯 权限分级规则

### 🟢 低风险操作（AI 直接执行，无需确认）

| 类别 | 允许的命令/操作 |
|------|----------------|
| **读取** | `cat`, `read`, `less`, `head`, `tail` |
| **搜索** | `grep`, `find`, `ls`, `dir` |
| **系统状态** | `df`, `free`, `top`, `ps`, `uptime`, `uname` |
| **创建** | `echo`, `write`, `mkdir`, `touch` |
| **下载** | `wget` (仅限已知安全 URL) |
| **分析** | `pdf`, `image`, `web_fetch` |

**规则：** AI 可以直接执行，事后在操作日志中记录

---

### 🟡 中风险操作（AI 申请 + 用户确认）

| 类别 | 需要确认的操作 | 确认提示模板 |
|------|---------------|--------------|
| **删除** | `rm`, `rmdir`, `trash` | "我想删除 [文件路径]，原因是 [原因]。确认吗？" |
| **修改** | `edit`, 修改现有文件 | "我想修改 [文件]，改动是 [内容]，原因是 [原因]。确认吗？" |
| **安装** | `apt install`, `pip install`, `npm install` | "我想安装 [软件]，用于 [用途]。确认吗？" |
| **服务** | `systemctl start/stop/restart` | "我想 [操作] [服务]，原因是 [原因]。确认吗？" |
| **配置** | 修改 `/etc/` 下文件 | "我想修改 [配置]，影响是 [影响]。确认吗？" |
| **权限** | `chmod`, `chown` | "我想修改 [文件] 权限为 [权限]。确认吗？" |
| **网络** | 开放/关闭端口 | "我想 [开放/关闭] 端口 [端口号]，用于 [用途]。确认吗？" |

**规则：** AI 必须先说明"做什么 + 为什么"，用户回复"确认"后才执行

---

### 🔴 高风险操作（AI 只给指令，用户手动执行）

| 类别 | 操作 | AI 响应 |
|------|------|--------|
| **执行脚本** | 运行未知脚本 | "这个操作有风险，请手动执行：[命令]" |
| **系统核心** | 修改核心配置 | "请手动编辑 [文件]" |
| **root 权限** | 需要 sudo 的操作 | "请用 sudo 执行：[命令]" |
| **密码凭证** | 修改密码/密钥 | "请手动修改 [密码/密钥]" |
| **重装系统** | 重置/初始化 | "请在 [控制台] 执行 [操作]" |
| **物理操作** | 硬件相关 | "请手动 [操作]" |

**规则：** AI 只提供指令和说明，不执行

---

## 📋 确认流程

### 标准确认流程

```
AI: "我想执行 [操作]，原因是 [原因]，影响是 [影响]。确认吗？"
用户: "确认" → AI 执行
用户: "不确认" → AI 不执行
用户: "为什么？" → AI 解释 → 用户再确认
```

### 快速确认（可配置）

对于重复性操作，用户可以授权"本次会话不再确认同类操作"

---

## 🛡️ 安全边界

### 绝对禁止的操作（即使确认也不执行）

| 操作 | 原因 |
|------|------|
| 删除系统核心文件 (`/bin`, `/usr`, `/etc` 关键文件) | 可能导致系统无法启动 |
| 格式化磁盘 | 数据丢失风险 |
| 修改 root 密码 | 可能导致无法登录 |
| 刷写 BIOS/固件 | 硬件损坏风险 |
| 发送敏感数据到外部 | 数据泄露风险 |

